# Deployment

BlockNot is deployed using [Kamal 2](https://kamal-deploy.org/) to a Hetzner VPS.

## Server Setup (Hetzner)

### 1. Create a VPS

1. Go to [Hetzner Cloud Console](https://console.hetzner.cloud/)
2. Create a new project
3. Add a server:
   - **Image:** Ubuntu 22.04
   - **Type:** CX22 (2 vCPU, 4 GB RAM) — sufficient for a small group
   - **Location:** Choose closest to your users
   - **SSH Key:** Add your public key

### 2. Point your domain

Add an A record pointing to your server IP:

```
chat.example.com → 123.45.67.89
```

### 3. Initial server setup

```bash
ssh root@your-server-ip

# Update packages
apt update && apt upgrade -y

# Install Docker (Kamal requirement)
curl -fsSL https://get.docker.com | sh
```

## Kamal Configuration

### Install Kamal locally

```bash
gem install kamal
```

### Configure `config/deploy.yml`

```yaml
service: blocknot

image: your-dockerhub-username/blocknot

servers:
  web:
    hosts:
      - your-server-ip
    labels:
      traefik.http.routers.blocknot.rule: Host(`chat.example.com`)
      traefik.http.routers.blocknot.tls.certresolver: letsencrypt

registry:
  username: your-dockerhub-username
  password:
    - KAMAL_REGISTRY_PASSWORD

env:
  clear:
    PHX_HOST: chat.example.com
    PORT: 4000
    POOL_SIZE: 10
    MIX_ENV: prod
  secret:
    - DATABASE_URL
    - SECRET_KEY_BASE
    - TELEGRAM_BOT_TOKEN
    - TELEGRAM_BOT_USERNAME

traefik:
  options:
    publish:
      - "443:443"
      - "80:80"
    volume:
      - "/letsencrypt:/letsencrypt"
  args:
    entryPoints.web.address: ":80"
    entryPoints.websecure.address: ":443"
    entryPoints.web.http.redirections.entryPoint.to: websecure
    certificatesResolvers.letsencrypt.acme.email: your-email@example.com
    certificatesResolvers.letsencrypt.acme.storage: /letsencrypt/acme.json
    certificatesResolvers.letsencrypt.acme.httpchallenge.entrypoint: web

accessories:
  db:
    image: postgres:15
    host: your-server-ip
    port: 5432
    env:
      clear:
        POSTGRES_DB: blocknot_prod
      secret:
        - POSTGRES_PASSWORD
    directories:
      - data:/var/lib/postgresql/data
```

### Set secrets

```bash
# Set secrets that Kamal will inject at deploy time
kamal env push \
  DATABASE_URL="ecto://postgres:YOUR_DB_PASSWORD@blocknot-db:5432/blocknot_prod" \
  SECRET_KEY_BASE="$(mix phx.gen.secret)" \
  TELEGRAM_BOT_TOKEN="your-bot-token" \
  TELEGRAM_BOT_USERNAME="your_bot_username" \
  POSTGRES_PASSWORD="YOUR_DB_PASSWORD" \
  KAMAL_REGISTRY_PASSWORD="your-dockerhub-password"
```

## Dockerfile

The project uses a multi-stage Dockerfile (generated by `mix phx.gen.release --docker`):

```dockerfile
# Build stage
ARG ELIXIR_VERSION=1.16.1
ARG OTP_VERSION=26.2.1
ARG DEBIAN_VERSION=bookworm-20240130-slim

ARG BUILDER_IMAGE="hexpm/elixir:${ELIXIR_VERSION}-erlang-${OTP_VERSION}-debian-${DEBIAN_VERSION}"
ARG RUNNER_IMAGE="debian:${DEBIAN_VERSION}"

FROM ${BUILDER_IMAGE} as builder

ENV MIX_ENV="prod"

RUN apt-get update -y && apt-get install -y build-essential git nodejs npm \
    && apt-get clean && rm -f /var/lib/apt/lists/*_*

WORKDIR /app

COPY mix.exs mix.lock ./
RUN mix deps.get --only prod
RUN mkdir config
COPY config/config.exs config/runtime.exs config/prod.exs config/
RUN mix deps.compile

COPY priv priv
COPY lib lib
COPY assets assets
RUN mix assets.deploy
RUN mix compile
RUN mix release

# Runner stage
FROM ${RUNNER_IMAGE}

ENV MIX_ENV="prod"

RUN apt-get update -y && apt-get install -y libstdc++6 openssl libncurses5 locales ca-certificates \
    && apt-get clean && rm -f /var/lib/apt/lists/*_*

RUN sed -i '/en_US.UTF-8/s/^# //g' /etc/locale.gen && locale-gen
ENV LANG en_US.UTF-8

WORKDIR /app
RUN chown nobody /app
COPY --from=builder --chown=nobody:root /app/_build/prod/rel/blocknot ./

USER nobody

CMD ["/app/bin/server"]
```

## Deploy

```bash
# First deploy — sets up Traefik, database, and app
kamal setup

# Subsequent deploys
kamal deploy

# Check logs
kamal app logs

# Run migrations
kamal app exec "bin/blocknot eval 'BlockNot.Release.migrate()'"

# Open a remote console
kamal app exec -i "bin/blocknot remote"
```

## SSL/TLS

SSL is handled automatically by Traefik + Let's Encrypt:

- Traefik listens on ports 80 and 443
- HTTP requests are redirected to HTTPS
- Certificates are obtained and renewed automatically via ACME HTTP challenge
- Certificates are stored in `/letsencrypt/acme.json` on the host

No manual certificate management needed.

## Database Backups

### Manual backup

```bash
kamal accessory exec db "pg_dump -U postgres blocknot_prod" > backup_$(date +%Y%m%d).sql
```

### Automated daily backup with cron

On the server:

```bash
# Edit crontab
crontab -e

# Add daily backup at 3 AM
0 3 * * * docker exec blocknot-db pg_dump -U postgres blocknot_prod | gzip > /backups/blocknot_$(date +\%Y\%m\%d).sql.gz
```

### Restore from backup

```bash
cat backup_20240115.sql | kamal accessory exec db "psql -U postgres blocknot_prod"
```
